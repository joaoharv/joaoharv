## About Me

I work at the intersection of **AI engineering**, **machine learning** and **data engineering** — building systems that move from raw data to deployed, usable solutions.

My focus is not just on training models or running experiments in isolation. I care about the full stack: how data is ingested and transformed, how models are structured and validated, how intelligence is exposed through APIs, and how all of that connects to something that actually works in practice.

I have hands-on experience with **LLMs and autonomous agents**, building local inference systems with Llama 3.1 and Ollama, designing async FastAPI backends, and structuring ETL pipelines with PostgreSQL and Pandas. I think in terms of **end-to-end solutions** — not disconnected pieces.

Currently pursuing a degree in **Data Science and Artificial Intelligence at FIAP**, and constantly developing practical depth in MLOps, cloud architecture and generative AI systems.

## What I Build

I gravitate toward problems that require connecting multiple layers — data, models, APIs and business logic — into something coherent and maintainable.

```
  Raw Data  →  Engineered Features  →  Trained Model  →  API Layer  →  Real Usage
      │                │                     │                │               │
   ETL/SQL         Pandas/EDA           Scikit-learn      FastAPI        Value
   MongoDB         Feature Eng.         Validation        Async          Impact
   PostgreSQL      NLP/ML               Deployment        REST           Insight
```

In practice, this means building **ML pipelines** with proper validation and reproducibility, deploying **LLM-powered agents** that reason across multi-step tasks, structuring **data pipelines** that go from messy inputs to analytics-ready outputs, and exposing everything through **well-designed APIs** that others can actually use.

## AI Engineering Mindset

There is a meaningful difference between running a model and building an AI system. The former produces a result. The latter produces something you can maintain, scale, and rely on.

```
┌──────────────────────────────────────────────────────────────┐
│                     AI SYSTEM ARCHITECTURE                   │
├─────────────────┬───────────────────┬────────────────────────┤
│   Data Layer    │    Model Layer    │   Application Layer    │
│─────────────────│───────────────────│────────────────────────│
│ ETL Pipelines   │ LLMs (Llama 3.1) │ FastAPI (async)        │
│ Data Modeling   │ Scikit-learn      │ REST Endpoints         │
│ Feature Eng.    │ Autonomous Agents │ API Integration        │
│ Data Quality    │ Prompt Eng.       │ Error Handling         │
│ SQL / NoSQL     │ NLP / Embeddings  │ Typed Schemas          │
└─────────────────┴───────────────────┴────────────────────────┘
```

Running LLMs locally with Ollama is a deliberate architectural choice — not a workaround. It means full control over the inference stack, no external API latency, no per-token cost at scale, and no data leaving the environment. That kind of thinking — where infrastructure decisions are made with intention — is what separates systems from experiments.
